# Content Flywheel Production — Writer Perspective

What Writer produces within the flywheel and how each piece connects to the whole.

## Derivative Chain — Writer's Role

From one anchor piece per week, Writer produces these derivatives:

1. **X thread** (5-7 tweets) — extract key insight, expand into thread narrative
2. **TikTok caption** — first 80 chars = hook, narrative structure (never listicle in caption), 3-5 hashtags
3. **Instagram Reel caption** — first 125 chars visible, keyword-first line, 5 hashtag hard cap
4. **LinkedIn post** — professional framing, hook-first 2-liner, analytical angle
5. **Anchor video outline** — structured outline for human recording, NOT a script
6. **Build diary entry** — raw, honest, failures > milestones
7. **Deep post building blocks** — raw materials from anchor decomposition (paradoxes, arcs, quotables)
8. **Carousel content brief** — structured brief for Publisher → Coder rendering chain

Each derivative references the anchor but stands alone — a reader on any single platform gets full value without needing the others.

## Content Bucket Detail

### Tool Breakdown
- Tone: authoritative, specific, enthusiastic about the craft (not the hype)
- Structure: problem → tool → specific workflow → result with numbers
- Must name exact tools, versions, configs. "AI tool" is never acceptable.
- Angle: practitioner sharing what works, not reviewer ranking products

### Build-in-Public Update
- Tone: honest, reflective, slightly vulnerable
- Structure: what happened → what broke → what I learned → what's next
- Failures are more engaging than milestones. Lead with what went wrong.
- Numbers required: hours spent, lines of code, users gained, money burned

### Practitioner News Framing
- Tone: analytical, opinionated, grounded in builder experience
- Structure: what happened → why it matters for builders → what to do about it
- Always include a "so what" for the reader's own projects
- Never just report news — filter through practitioner lens

## Platform Constraints

Cross-reference `get_style_instructions()` for mechanical details. Key writer-relevant constraints:

- **TikTok caption**: 4000 chars max, first 80 visible before expand. On-screen text indexed for search — what you write in the caption should complement, not duplicate, on-screen text.
- **Instagram Reel**: 2200 chars, first 125 visible. Keywords in first line outperform hashtags for discovery.
- **Instagram Carousel**: 2200 char caption, 2-20 slides. First slide is hook. Save-optimized format.
- **X/Twitter**: 280 chars per tweet. Thread format for longer content. Hook in first tweet.
- **LinkedIn**: 3000 chars. Hook-first 2-liner before "see more" fold.

## Derivative Relationships

**Share core copy between:** X thread → LinkedIn post (same insight, different framing). TikTok caption → Reel caption (similar but different hashtag sets and visible-text optimization).

**Need fresh angles for:** Build diary (personal, reflective) vs. tool breakdown (authoritative, specific). Carousel brief (visual-first thinking) vs. thread (narrative-first).

**Sequence matters:** Anchor outline first → derivatives after. Deep post building blocks can inform multiple derivatives. Carousel brief needs all content decisions made before handoff to Publisher.

## Self-Check Quality Signals

Before marking any derivative complete:

- **Specificity**: Does it name at least one specific tool, number, or concrete example?
- **Hook strength**: Would the first line/sentence make someone stop scrolling? Test: cover everything except the first line — does it create curiosity?
- **CTA presence**: Is there a clear next action for the reader? (save, comment, follow, try it)
- **Standalone value**: Can someone on this platform alone get full value without seeing the anchor?
- **Voice match**: Does it sound like the social voice profile, not the analytical voice?
